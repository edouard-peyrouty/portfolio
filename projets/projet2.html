Automatisation de la chaine de production CI/CD

Pipeline CI/CD automatisée avec GitHub Actions et Docker, déclenché à chaque commit, assurant le build, la publication automatique des images Docker sur Docker Hub et un déploiement rapide grâce à un script d’exécution. Projet implémenté sur une application simple de test développée en python Flask avec une base de données mySQL.

Schéma de la pipeline

A chaque commit du développeur, gitHub Actions déclenche la pipeline CI/CD. Cette dernière construit les images Docker de l’application et de la base de données, les publie sur Docker Hub, puis un script d’exécution permet de récupérer la version la plus récente de l'application et de la lancer rapidement

Installation et lancement de l'application côté utilisateur

L'installation et l'exécution de l'application ne nécessitent aucune configuration complexe pour l'utilisateur. Il lui suffit de télécharger le script de lancement qu'il exécute pour lancer l'application. Lors de son exécution, le script récupère les images Docker les plus récentes depuis le registre, garantissant que l'utilisateur exécute toujours la dernière version stable de l'application.

Historique des exécutions sur GitHub Actions

Chaque commit sur la branche principale déclenche automatiquement la pipeline définie par un fichier yml. L'onglet Actions de GitHub affiche l'historique complet des exécutions avec leur statut, leur durée et le commit associé, permettant de suivre en temps réel le bon déroulement de chaque déploiement.

Hébergement d'images du Docker Hub

À chaque exécution réussie de la pipeline, les images Docker buildées sont automatiquement publiées sur Docker Hub. Le registre centralise les images des différents services de l'application, chacune versionnée avec le tag latest pointant toujours vers le build le plus récent.
